From Chaos to Composition: Building a Cellular Automata Music Generator with Web Audio

A deep dive into emergent musical patterns and the surprising musicality of simple rules

What if music could emerge from the same mathematical principles that govern the growth of snowflakes, the spread of forest fires, and the patterns on seashells? This is the premise behind cellular automata composition — a technique that uses simple, deterministic rules to generate complex, often beautiful musical sequences.

In this post, I'll walk through the development of a real-time cellular automata music generator built entirely with vanilla JavaScript and the Web Audio API. No external libraries, no MIDI files — just code, rules, and emergent sound.

What Are Cellular Automata?

Cellular automata (CA) are discrete computational systems consisting of a grid of cells, each in one of a finite number of states. At each time step, every cell updates its state based on the states of its neighbors according to fixed rules. The most famous example is John Conway's Game of Life, where cells live, die, or are born based on how many neighbors surround them.

The key insight that makes CA musically interesting is this: simple local rules can produce globally complex, evolving patterns. A CA that starts from random noise will often settle into rhythmic cycles, traveling waves, or stable structures — all properties we associate with music.

The Mapping Problem: From Pixels to Pitches

The central challenge in CA composition isn't implementing the automaton itself — that's straightforward. The real puzzle is how to map the visual grid to musical parameters. This is where I discovered insights that went beyond what we covered in class.

My approach maps the CA grid as follows:

  • Rows → Pitches: Each row corresponds to a note in the chosen musical scale
  • Columns → Time: The playhead moves left-to-right across each generation
  • Cell states → Note triggers: Living cells trigger their row's note
  • Generations → Macro-time: After scanning one grid, we evolve to the next generation

What surprised me was how much the choice of scale affects the musical result. Running the exact same CA pattern through a pentatonic scale versus a chromatic scale produces radically different emotional content. The pentatonic scale's lack of semitones acts as a natural "consonance filter," making almost any random pattern sound pleasant. The chromatic scale, by contrast, exposes the CA's chaotic nature.

"The automaton is the composer; the scale is the editor."

Choosing Rules: Not All CA Are Musical

I implemented five different rule sets and found dramatic differences in their musical utility:

Game of Life (B3/S23) produces the most "flowing" compositions. Its balance of stability and chaos creates patterns that breathe — dense regions thin out, sparse regions fill in, creating natural dynamics.

Seeds (B2/S) is explosive. Every living cell dies immediately, so patterns cascade outward like fireworks. Musically, this creates rapid bursts of notes followed by silence — dramatic but exhausting over time.

Day & Night (B3678/S34678) surprised me the most. Its symmetry between birth and survival rules means dense and sparse regions are treated equivalently. This creates a kind of "negative space" music where the silence is as structured as the sound. 

// The rules that govern musical character
const RULE_PRESETS = {
    life:     { birth: [3],       survive: [2, 3] },      // Balanced, flowing
    seeds:    { birth: [2],       survive: [] },           // Explosive, sparse
    daynight: { birth: [3,6,7,8], survive: [3,4,6,7,8] }  // Symmetric, dense
};

The Web Audio Architecture

When multiple cells in a column are alive, we play multiple notes simultaneously — effectively creating chords. This posed a technical challenge: how do you prevent clipping when the density of living cells varies wildly?

My solution uses a DynamicsCompressor node as a limiter. This automatically reduces gain when the signal gets too hot, allowing dense moments to be loud-but-not-distorted: 

// Prevent clipping with automatic compression
this.compressor = this.audioContext.createDynamicsCompressor();
this.compressor.threshold.value = -24;  // Start compressing at -24dB
this.compressor.ratio.value = 12;       // Strong compression when triggered
this.compressor.attack.value = 0.003;   // Fast attack catches transients

Each note uses an ADSR envelope (Attack, Decay, Sustain, Release) to shape its amplitude over time. Without envelopes, notes would click harshly at the start and end. With them, notes blend smoothly into each other, creating legato passages when cells persist across generations.

Unexpected Discoveries

1. Initial density matters more than rules. A 20% initial density with Game of Life rules sounds completely different from 50% density with the same rules. Lower densities create sparse, contemplative music; higher densities create dense, overwhelming textures.

2. The toroidal topology creates loops. Because edges wrap around (what exits right re-enters left), patterns that would die at boundaries instead cycle back, creating unexpected repetition and structure.

3. Tempo changes the perceived genre. At 60 BPM, the output sounds ambient. At 180 BPM, the same pattern sounds like glitchy electronica. The CA doesn't know what tempo it's generating for — this interpretation layer is entirely artificial.

Future Directions

This implementation scratches the surface of what's possible. Some ideas for extension:

  • Use cell age (how long it's been alive) to modulate timbre
  • Map cell birth events to accented notes versus sustained cells to held notes
  • Use multiple CA running in parallel for different instruments
  • Implement 3D CA where the Z-axis controls another musical parameter (e.g., filter cutoff)

Conclusion

Building this system taught me that automated composition is less about algorithms finding good music and more about designers creating constraints that make bad music unlikely. The cellular automaton doesn't "know" anything about music theory — it just follows rules. But by carefully choosing the mapping (scale, tempo, waveform, envelope), we can ensure that whatever emerges will at least be interesting.

This is perhaps the deepest lesson: in generative music, the art isn't in the algorithm. It's in the space of possibilities you allow the algorithm to explore.
